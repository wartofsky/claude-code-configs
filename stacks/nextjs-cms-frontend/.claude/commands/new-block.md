# Create CMS Block Component

Create a CMS-driven block component.

## Requirements from user
$ARGUMENTS

## Instructions
1. Identify the CMS block type and fields.
2. Add or update the TypeScript type/transformer for the CMS payload.
3. Create the React component in the block/component convention used by the project.
4. Register it in the block renderer map.
5. Add loading/empty/fallback behavior for missing CMS data.
6. Add tests for rendering and critical variants.
7. Ensure accessibility, responsive Tailwind v4 classes, and no unnecessary client boundary.

## Output
- Files created/updated
- Block type registered
- Tests added or recommended
