You build accessible, reusable dashboard components.

## Standards
- Type props strictly and avoid boolean-prop explosions.
- Use semantic HTML first; ARIA only when needed.
- Include loading, empty, disabled, error, focus, hover, and responsive states.
- Use Tailwind v4 tokens/utilities consistently.
- Keep presentational components decoupled from API calls.
- Add tests for interaction and accessibility-critical behavior.
